# 翻译 Skill 参考说明

## 版权与转载默认策略

- 翻译前确认原文许可：常见如 CC 系列需保留署名与许可证链接；「All Rights Reserved」仅翻译摘要或引导至原文，除非已获得书面授权。
- 在「转载与版权」中写明：原文许可证名称、链接、是否修改、译者与翻译日期。
- 不确定时：停止全文翻译，仅输出摘要 + 原文链接，并提示用户核实授权。

## 专有名词（示例，非穷尽）

- 保留英文：`OpenAI`、`GPT-4`、`Kubernetes`、`CUDA`、`RAG`、`LoRA`、`MIT License`。
- 可首次括注：`检索增强生成（RAG）` 后文可用 `RAG`。
- 人名：一般保留原文拼写；广为人知者可按通行中文译名。

## `topics` 与 `original_site` 命名

- `topics`：技术主题，小写英文单词或连字符短语，如 `llm`、`agent`、`security`、`claude-code`、`responsive-design`；3-6 个为宜，不超过 8 个。翻译前先用 `rg "topics" content/translations/ --no-filename` 查看已有值，优先复用。
- `original_site`：人类可读的站点品牌名，同一站点多篇文章保持一致，便于列表「来源」筛选聚合。

## `translation_category` 命名规范

- **长度**：2-4 个中文字（或广为人知的英文缩写如 `DevOps`）。
- **面向读者**：普通技术读者能直觉理解其含义。
- **不重复**：新分类在语义上不应与已有分类高度重叠（如不要同时存在「前端」和「前端开发」）。

### 已有分类与 CSS 颜色清单

新增分类时从下方「备用色板」中选取，追加到 `assets/css/extended/translations.css` 分类颜色规则区域。格式为：

```css
.translations-badge-category[data-category="分类名"] {
    color: <色板主色>;
    background: rgba(<R>, <G>, <B>, 0.1);
}
```

### 备用色板（用于新增分类）

| 色名 | hex | rgba 前缀 |
|------|-----|-----------|
| rose | `#e11d48` | `225, 29, 72` |
| lime | `#84cc16` | `132, 204, 22` |
| cyan | `#06b6d4` | `6, 182, 212` |
| fuchsia | `#d946ef` | `217, 70, 239` |
| amber-dark | `#b45309` | `180, 83, 9` |
| slate | `#475569` | `71, 85, 105` |

若备用色也用尽，可从 Tailwind CSS 调色板的 500 级色阶中选取，保持 `0.1` 不透明度的 background。

CSS 基础规则已有默认回退色（`var(--secondary)`），新分类即使暂未添加专属颜色也不会显示异常。但为了视觉一致性，建议始终补上。
